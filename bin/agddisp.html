Name

AGDDISP: AGD File Display Application

Software Structure

AGDDISP.EXE: The Program
DISP.BIN: Part executed on the PC-FXGA

Syntax

AGDDISP AGD file name [ATT file name] [ACD file name]

Functionality

Reads the AGD file (input_file.agd) and the ATT file (att_file.att), as well as the ACD file (acd_file.acd) and displays it on the actual device. If the ATT file is omitted, it searches for input_file.att and reads that. If the ACD file is not specified, it does not read it. For each file, the extension (.agd/.att/.acd) may be omitted, but in that case it will be interpreted as the first being the AGD file, the second the ATT file, and the third the ACD file. If the extension is specified for the ATT file or the ACD file, the order can be changed, or one may be omitted.

ATT File Compatibility

If the ATT file is not specified, the program will read infile.att. In this case, the ATT file will be opened as if it is in the same directory as infile. If you would like to read a different file as the ATT file, it must be specified. Next, the AID file listed in the ATT file is read. If the file does not exist, it does not read it. The texture data will be loaded into the texture bank listed in the AID file. To change the load destination of the texture data, the AID file must be changed. If the Texture_tag number listed in the AGD file is also listed in the ATT file, the corresponding bank’s textures are used for the display. Furthermore, the UV coordinates are recalculated based on the location and size listed in the ATT file.

Prioritization in IC Mode

The IC mode specified in each file is prioritized in the order of the ACD file, the AID file, then the AGD file. If the IC mode is not the same across the files, there will not be an error, but the desired results may not be displayed.

Color Map

When an ACD file is specified, the contents of the ACD file will be used as the color map. Otherwise, the default color map for the IC mode is set.
