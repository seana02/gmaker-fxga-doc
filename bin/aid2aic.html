Name

AID2AIC: AID→AIC Converter

Syntax

AID2AIC [Switch] AID File name [AIC file name]

Functionality

Converts AID Files to AIC files in command packet format. The first 4 bytes of an AIC file is the size of the packet data, then the data follows. The following formats can be converted:

  • AID_IMG_FORMAT_INDEX9
  • AID_IMG_FORMAT_INDEX12
  • AID_IMG_FORMAT_INDEX16
  • AID_IMG_FORMAT_UNPACK12
  • AID_BG_FORMAT_INDEX9
  • AID_BG_FORMAT_INDEX12
  • AID_BG_FORMAT_INDEX16
  • AID_BG_FORMAT_UNPACK12
The data in AID_IMG_FORMAT_INDEX9, AID_BG_FORMAT_INDEX9 are unpacked based on the I-C mode in the header before converting. Also, in the current version, AID_IMG_FORMAT_INDEX12, AID_BG_FORMAT_INDEX12, AID_IMG_FORMAT_FORMAT12, AID_BG_FORMAT_FORMAT12 are treated the same.

Switches

-x(-binary)
Outputs a binary file.

-a(-ascii)
Outputs a text file. The format is hexadecimal with a leading 0x.

-b(-byte)
When outputting a text file, outputs in bytes.

-h(-half)
When outputting a text file, outputs in half words.

-w(-word)
When outputting a text file, outputs in words.

-C
Outputs in C Array format.

-S
Outputs in assembler data definition format.

-banksel
Outputs code to choose the bank to house texture data.

-nobanksel
Doesn't output code to choose the bank to house texture data.

-beef
Outputs a command with a termination word.

-nobeef
Outputs a command without a termination word.

-forcei6c3
Outputs with IC Mode I=6, C=3, ignoring color information for the image data.

-forcei5c4
Outputs with IC Mode I=5, C=5, ignoring color information for the image data.

-forcei4c5
Outputs with IC Mode I=4, C=5, ignoring color information for the image data.

-forcei3c6
Outputs with IC Mode I=3, C=6, ignoring color information for the image data.

-putdisp
Outputs code that write image data to the frame buffer rather than the texture buffer.

-z num
Set the Z value used with the -putdisp switch.
